Here are some projects I am currently working on.

The great Solovay-Kitaev word search

In classical computing there is a theorem that you can build any circuit from a finite set of “universal” gates. These gates are universal in the sense that any circuit can be composed of just gates from this set, for example {${\text{AND}, \text{NOT}}$} is commonly taken as a universal and complete set of gates from which all others can be derived. In quantum computing you are allowed a continuum of gates, in general you consider elements of the compact Lie group $SU(n)$ which are $n \times n$ unitary matrices of determinant 1. A concrete example of a continuous family of gates are rotations by some angle $\theta$ where $\theta$ varies continuously. The Solovay-Kitaev theorem is a fundamental theorem of quantum computing which asserts that given an inverse-closed subset $\mathcal{G}$ of $SU(n)$ where $\mathcal{G}$ densely generates the Lie group, you can construct a sequence of elements (also called words) in $\mathcal{G}$ which can approximate any $U \in SU(n)$. If $S$ is such a word then we have an important result that for some precision $\varepsilon$ we can achieve $||S - U|| < \varepsilon$ with a word length of $O((\log 1 / \varepsilon)^{\alpha})$ where $\alpha > 3$. In other words, the exponent $\alpha$ directly corresponds to compilation overhead for all your quantum operations.

The point of this project is to create a high performance algorithm which finds approximating words to assist in an improved Solovay-Kitaev algorithm.

Complexity zoology

Continuing previous work on the complexity zoology project, an expert system that is meant to reason about complexity classes. This project relates classical and quantum complexity classes as well as their relativizations (as in, with respect to some oracle). This work has lead to the discovery of new open problems in complexity theory and serves as a quick reference for complexity theorists to learn the status of their favorite complexity classes. Some goals for this project are to include more relationships with respect to algebraic oracles, a newer and important type of separation outlined in Aaronson and Wigderson, as well as clean up and correct existing work done on the project. Finally, the goal is to add new inclusions and more complexity classes to give a better overall view of the “complexity zoo”.
